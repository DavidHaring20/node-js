Res.send() produces errors about headers

You are here because you probably got something like this Cannot set headers after they are sent to the client error. That happens because you have sent 2 times in the same app.get() method. One of the probable errors could lie in your IF statement.
Example of an IF statements: 

                if (condition) {
                    res.send(1);
                }
                res.send(2);

If the condition is true, the the res.send(1) will be executed. And as the compiler goes towards bottom it will executed res.send(2). If that happens you will get error and a crash.
Much wiser approach would be to ensure that if the condition is true that it only executes res.send(1).
Example: 

                if (condition) {
                    return res.send(1);
                }
                res.send(2);

What is "req.query" and what it consists of ?

"req.query" is a property that is an object containing a property for each query string parameter in the route. When QUERY PARSER is set to disabled, it is an empty object {}, otherwise it is the result of the configured query parser. Each query object is composed of two elemements:
- key
- value
In the following example:
localhost:8080/search?q=tobbi+ferret
"search" is a key, and tobbi+ferret is a value.

express.static(root, [options])

It is a built in middleware function in Express. It serves static files and is based on "serve-static"(an npm module). The root argument specifies the directory from where it should serve static files. Let's say that we have a directories called: "assets" for pictures and "styles" for CSS stylesheets. If we would need the server to serve them then we need to configure it like the following: 

                app.use(express.static('assets'));
                app.use(express.static('styles'));

Client server model

Client server model describes how server provides resources to one or more clients. Types of servers: web servers, mail servers and file servers. Types of clients: desktop computers, laptops, tablets and smartphones.

Content Delivery Network (CDN)

Content Delivery Network is a geographically distributed network of proxy servers and their data centers. The goal is to provide high availability and performance by distributing the service spatially relative to end users.
